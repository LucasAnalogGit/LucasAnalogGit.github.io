# 基于 TPE 的模拟电路参数优化方法

## 1. 背景：为什么不能靠暴力搜索调电路

在前几篇文章中，我们已经把模拟 IC 自动化设计的执行链路搭了起来：Python 可以连接 Virtuoso，SKILL 可以生成 schematic，Spectre/OCEAN 可以自动仿真，Python 也可以从结果中提取 Gain、GBW、PM、SR、Power 等性能指标。上一篇文章进一步把这个流程抽象成黑箱优化问题：

```text
设计参数 x
        ↓
Spectre 黑箱仿真
        ↓
性能指标 metrics
        ↓
损失函数 L(x)
```

到这里，真正困难的问题就出现了：优化器到底应该如何选择下一组 MOS 尺寸、偏置电流和补偿参数？

如果只是随机尝试，仿真次数很快就会失控。如果做网格搜索，问题会更夸张。假设一个简单 OTA 有 10 个可调参数，每个参数只取 10 个候选值，那么总组合数就是：

```text
10^10
```

这意味着一百亿次候选点。即使每次 Spectre 仿真只花几十秒，这个数量级也完全不可接受。更现实的情况是，很多 trial 还会失败：DC 不收敛、输出饱和、相位裕度无法提取、某个波形不存在，或者指标变成 `NaN`。

这也是为什么模拟电路自动优化的关键，不是“能不能跑仿真”，而是“如何少跑一些无意义的仿真”。TPE 的价值就在这里。它不是盲目搜索，而是根据已经跑过的仿真结果，逐步判断哪些参数区域更值得继续尝试。

## 2. 从黑箱优化到 Bayesian Optimization

### 2.1 模拟电路优化为什么像一个黑箱

直观地看，模拟电路优化很像在调一个复杂机器。你给它一组参数，例如输入管宽度、负载管尺寸、偏置电流、补偿电容；它经过 Virtuoso/Spectre 仿真后，返回一组指标。你能观察输入和输出，但很难写出一个简单解析公式直接描述中间关系。

数学上，可以把这个过程写成：

```text
m = F(x)
L = E(m)
```

其中：

- `x` 是设计参数；
- `F(x)` 是由 schematic、netlist、Spectre、OCEAN 和结果提取脚本组成的黑箱仿真函数；
- `m` 是仿真指标向量；
- `E(m)` 是评价函数，把多项指标转化为单个 loss。

在模拟电路中，这意味着优化器并不知道“增大某个 MOS 宽度后 GBW 会如何变化”的解析规律。它只能不断提出参数、运行仿真、观察结果，然后根据历史经验调整下一次采样。

### 2.2 Bayesian Optimization 的直观理解

Bayesian Optimization 的直观思想是：与其盲目搜索整个参数空间，不如根据已经评价过的样本建立一个概率模型，用它估计哪些区域更可能产生更好的结果，然后优先去这些区域继续仿真。

历史样本可以写成：

```text
D_n = {(x_1, y_1), (x_2, y_2), ..., (x_n, y_n)}
```

其中：

- `x_i` 是第 `i` 次 trial 的设计参数；
- `y_i` 是对应目标值，例如 loss；
- `D_n` 是已经完成的仿真历史。

优化目标则是：

```text
x* = argmin L(x)
```

在模拟电路优化中，这句话的工程含义是：在有限仿真预算内，尽快找到满足 Gain、GBW、PM、SR、Power 等指标的参数组合，而不是把所有可能的 W/L 组合都试一遍。

### 2.3 代理模型与采集函数

Bayesian Optimization 通常由两部分组成：

| 部分 | 作用 |
|---|---|
| 代理模型 | 根据历史样本估计目标函数的分布 |
| 采集函数 | 决定下一次应该评价哪个候选点 |

直观地说，代理模型负责“总结经验”，采集函数负责“决定下一步去哪”。代理模型不需要完全准确，它只需要比盲目搜索更会利用历史信息。

在模拟电路中，这一点很重要。每一个 `x_k` 都可能意味着：

```text
更新参数 → 生成 netlist → Spectre 仿真 → 读取结果 → 计算 loss
```

因此，采集函数的目标不是找一个理论上完美的点，而是在工程预算内优先选择更值得仿真的候选点。

### 2.4 Exploration 与 Exploitation 的平衡

优化器有两个相互竞争的需求：

- `Exploration`：探索还不熟悉的区域，避免过早陷入局部最优；
- `Exploitation`：利用已有好结果，在看起来有希望的区域继续细化。

如果只 exploitation，优化器可能很快围着一个不够好的局部区域打转。如果只 exploration，又会像随机搜索一样浪费仿真预算。

在模拟电路中，这个平衡很实际。比如某一批参数让 GBW 很高，但功耗也很高；另一批参数功耗低，但相位裕度不足。优化器需要继续探索两类区域附近的可能性，而不是过早判定某一类方向一定更好。

## 3. TPE 算法的基本思想

### 3.1 从“预测结果”到“总结好参数”

传统 Gaussian Process 风格的 Bayesian Optimization 常见做法是建立：

```text
p(y | x)
```

也就是问：“给定这组参数，结果可能是多少？”

TPE 的关键思路是反过来建模：

```text
p(x | y)
```

也就是问：“如果结果很好，那么参数通常长什么样？”

直观地说，TPE 像一个会总结经验的优化器。它观察历史 trial 中哪些参数组合带来了较低 loss，然后估计“好设计通常长什么样”；同时它也估计“普通或较差设计通常长什么样”。下一次采样时，它更倾向于选择那些像好设计、又不像差设计的参数组合。

在模拟电路优化中，这种思路很自然。优化器不需要真正理解“为什么某个补偿电容让 PM 变好”，它只需要从历史样本中发现：哪些 `Cc`、`W`、`L`、`m`、`Ibias` 区域更常出现在低 loss 设计中。

### 3.2 优秀样本与普通样本的划分

TPE 会根据历史目标值把样本分成两类。对于最小化问题，loss 较小的一部分被视为“好样本”，其余样本被视为“普通或较差样本”。

可以用一个阈值 `y*` 表示分界：

```text
y < y*      好样本
y ≥ y*      普通或较差样本
```

这里的 `y*` 通常由分位数决定，例如取历史样本中表现较好的一部分。它不是某个电路指标规格，而是优化器为了区分历史样本质量而设置的统计阈值。

在模拟电路中，“好样本”不一定已经完全满足所有指标。早期 trial 里，可能没有任何一个设计真正可行。即便如此，TPE 仍然可以利用“相对没那么差”的样本，逐步判断哪些参数区域更有希望。

### 3.3 l(x) 与 g(x) 的含义

TPE 的核心表达式是：

```text
l(x) = p(x | y < y*)
g(x) = p(x | y ≥ y*)
```

直观解释：

- `l(x)` 表示好样本对应的参数分布；
- `g(x)` 表示普通或差样本对应的参数分布。

数学上，TPE 不直接估计 `p(y | x)`，而是估计条件密度 `p(x | y)`。它把历史样本按目标值分开，再分别拟合两个参数分布。

落到模拟电路里，可以这样理解：如果某些输入管宽度、补偿电容和偏置电流组合经常出现在低 loss trial 中，那么这些区域会让 `l(x)` 更高；如果另一些组合经常导致不收敛、功耗过高或相位裕度不足，那么这些区域会让 `g(x)` 更高。

### 3.4 为什么选择 l(x)/g(x) 较大的候选点

TPE 倾向于选择：

```text
x_next = argmax l(x) / g(x)
```

更准确地说，实际实现中通常会先从 `l(x)` 中采样多个候选点，然后比较它们的 `l(x)/g(x)`，选择更有潜力的点，而不是在连续空间里精确求全局最大值。

直观解释是：一个好的候选点，应该“像历史好样本”，同时“不像历史差样本”。也就是 `l(x)` 高、`g(x)` 低。

在模拟电路中，这意味着 TPE 会逐步把搜索重点从完全随机的 W/L 组合，转向更可能满足 Gain、GBW、PM、SR 等指标的参数区域。但它并不保证全局最优，也不保证每次采样都比上一次更好。它只是更有策略地使用有限仿真预算。

## 4. TPE 的数学原理

### 4.1 历史 trial 数据

一次 TPE 优化可以看作不断扩展历史数据集：

```text
D_n = {(x_i, y_i)}_{i=1}^{n}
```

每完成一次 Spectre 仿真，就会得到一个新的 `(x_i, y_i)`。这里的 `y_i` 通常是 loss，而不是单独的 Gain 或 GBW。

在工程实现中，每个 trial 通常还会保存更多信息：仿真状态、metrics、是否可行、错误原因、运行时间、netlist 和日志路径等。TPE 主要使用目标值更新采样策略，但这些附加信息对调试和后续分析非常重要。

### 4.2 分位数阈值 y*

TPE 用阈值 `y*` 把历史 trial 分成两组：

```text
D_good = {(x_i, y_i) | y_i < y*}
D_bad  = {(x_i, y_i) | y_i ≥ y*}
```

直观上，`y*` 决定了“哪些样本算好”。如果这个阈值太严格，好样本太少，分布估计会不稳定；如果太宽松，好样本和普通样本差别不明显，采样方向也会变弱。

在模拟电路优化中，早期 trial 数量较少时，通常需要一定的随机或准随机启动样本。否则 TPE 没有足够历史信息，很难判断哪些参数区域值得信任。

### 4.3 条件概率密度建模

TPE 分别对好样本和普通样本建立条件概率密度：

```text
l(x) = p(x | y < y*)
g(x) = p(x | y ≥ y*)
```

对于连续变量，例如 MOS 宽度或补偿电容，可以用概率密度估计；对于整数变量和类别变量，例如 `m`、`nf` 或某种结构选项，也可以用相应的离散分布处理。

这也是 TPE 在工程上很实用的原因之一。模拟电路参数很少是纯连续变量：有些参数适合 log scale，有些必须是整数，有些只能从候选集合中选。TPE 比传统 GP 方法更容易接入这种混合搜索空间。

### 4.4 Expected Improvement 与采样准则

Expected Improvement 的直观意义是：在候选点 `x` 处，目标值比当前最好结果进一步降低的期望幅度。

对于最小化问题，可以写成：

```text
EI(x) = E[max(y_best - y(x), 0)]
```

其中 `y_best` 是当前历史中最好的 loss。如果某个候选点既可能明显优于当前最好结果，又有一定概率实现这种改进，那么它的 EI 就会较大。

TPE 的推导表明，选择较大的 `l(x)/g(x)` 与提高 EI 有关。因此，TPE 不必直接对 `p(y | x)` 建模，也能通过比较好样本分布和差样本分布来选择更有潜力的候选点。

在模拟电路中，这可以理解为：TPE 希望把仿真预算花在“有机会让 loss 继续下降”的区域，而不是把时间浪费在已经多次证明不收敛或性能很差的参数组合上。

## 5. 为什么 TPE 适合模拟电路参数优化

### 5.1 Spectre 仿真成本高

模拟电路中的一次函数评价往往不是一个简单函数调用，而是一整条流程：

```text
更新参数 → netlist → Spectre 仿真 → 读取结果 → 计算指标
```

如果电路复杂、仿真包含 AC 和 transient，单次 trial 的时间成本会进一步增加。TPE 的目标不是穷尽所有设计点，而是在有限仿真预算下更快找到有希望的区域。

### 5.2 目标函数不可导

模拟电路优化的目标通常可以写成：

```text
L(x) = penalty(Gain, GBW, PM, SR, Power)
```

这些指标来自仿真波形和工作点结果，不存在一个简单可用的解析梯度。即使某些局部敏感度可以通过电路知识估计，完整流程中的 netlist、仿真器、指标提取和失败处理也很难组成一个可微函数。

TPE 不要求梯度，这对模拟电路优化非常友好。

### 5.3 变量类型混合

模拟电路优化变量常常是混合类型：

| 变量 | 类型 |
|---|---|
| MOS 宽度 W | 连续变量 |
| MOS 长度 L | 连续或离散变量 |
| multiplier m | 整数变量 |
| finger number nf | 整数变量 |
| 补偿电容 Cc | 连续变量 |
| 零点电阻 Rz | 连续变量 |
| 补偿结构选择 | 类别变量 |

TPE 可以比较自然地处理这些变量。相比之下，传统 GP Bayesian Optimization 在高维、混合、条件搜索空间中往往需要更多工程处理。

### 5.4 搜索空间非凸且强非线性

模拟电路性能之间存在大量折中：

- 提高偏置电流可能增加 GBW，但也增加功耗；
- 增大补偿电容可能改善相位裕度，但降低带宽；
- 增大输入管尺寸可能提高 gm、降低噪声，但也增加寄生电容；
- 提高增益和提高速度往往存在冲突；
- 某些尺寸组合会让输出共模或工作区突然失效。

这些关系通常不是简单凸函数。TPE 虽然不能保证找到全局最优，但它能够根据历史结果逐步避开明显差的区域，并集中探索更有希望的区域。

### 5.5 可以处理失败 trial

在模拟电路优化中，失败 trial 是常态，而不是异常。某些参数组合可能导致：

- DC 不收敛；
- 输出节点饱和；
- 相位裕度无法提取；
- transient 波形不存在；
- 指标为 `NaN` 或 `inf`；
- 仿真日志中出现 fatal error。

工程上不应让这些 trial 直接中断整个 study。更稳妥的做法是记录失败原因，并返回一个较差 loss。例如：

```text
simulation_failed → loss = large_penalty
metric_invalid    → loss = large_penalty
precheck_failed   → loss = large_penalty
```

这样，TPE 也能从失败样本中学习：哪些参数区域不值得继续投入仿真预算。

### 5.6 适合逐步缩小有效设计区域

模拟电路优化通常不是一步到位。更常见的过程是：

1. 先找到能正常偏置、能仿真的区域；
2. 再找到满足主要指标的可行区域；
3. 最后在可行区域内压缩功耗、面积或过设计。

TPE 很适合这种逐步收缩的过程。它可以先通过随机启动和探索建立基本分布，再根据较好的 trial 逐步细化搜索。实际工程中，也可以把 TPE 与 gm-Id 初值、space-filling 初始采样、局部搜索或手工 seed 结合起来。

## 6. TPE 在自动化设计流程中的位置

把 TPE 放回整个自动化流程，它处在“黑箱建模”和“损失函数评价”之间：

```text
定义 search space
        ↓
TPE sampler 生成参数 x_k
        ↓
Python 写入设计变量
        ↓
SKILL / OCEAN / Spectre 执行仿真
        ↓
Python 提取 metrics
        ↓
计算 loss
        ↓
TPE 更新历史分布
        ↓
生成下一组参数 x_{k+1}
```

这里要注意一个关键点：TPE 只能看到 `loss`，它并不能直接理解电路。它不知道 Gain、GBW、PM、SR 各自的物理含义，也不知道某个 MOS 是否应该匹配。所有这些工程知识，都必须通过 search space、参数绑定、失败处理和 loss function 进入优化流程。

这也是为什么优化算法不能孤立讨论。一个好的 TPE 配置，背后一定有合理的参数空间、可靠的仿真 backend、可信的指标提取和清晰的 loss 设计。

## 7. 一个简化的 TPE 优化流程示例

下面是一个简化的 Optuna 风格伪代码，只用于说明流程，不对应任何真实工程实现。

```python
import optuna

def objective(trial):
    params = {
        "w_in": trial.suggest_float("w_in", 1e-6, 50e-6, log=True),
        "l_in": trial.suggest_float("l_in", 180e-9, 1e-6),
        "m_in": trial.suggest_int("m_in", 1, 16),
        "cc": trial.suggest_float("cc", 100e-15, 5e-12, log=True),
    }

    success = run_spectre_simulation(params)
    if not success:
        return 1e9

    metrics = extract_metrics()
    loss = compute_loss(metrics)
    return loss

sampler = optuna.samplers.TPESampler()
study = optuna.create_study(direction="minimize", sampler=sampler)
study.optimize(objective, n_trials=100)
```

这个例子中：

- `suggest_float` / `suggest_int` 定义搜索空间；
- `run_spectre_simulation()` 是高成本黑箱评价；
- `extract_metrics()` 将仿真波形转化为结构化指标；
- `compute_loss()` 将多指标转化为单个优化目标；
- `TPESampler` 根据历史 trial 更新采样策略。

在实际工程中，我更关心的不只是最后的 `best_value`，还包括每次 trial 的状态、参数、metrics、loss breakdown、错误原因和运行时间。因为优化失败时，最有价值的信息往往不在“最优点”，而在那些失败样本告诉我们的边界上。

## 8. 与模拟电路工程约束的结合

TPE 负责采样，但模拟电路工程约束不能指望 TPE 自己理解。它们需要通过流程设计显式进入优化器。

常见方式包括：

1. 搜索空间边界；
2. 参数绑定；
3. 参数离散化；
4. 失败 trial 惩罚；
5. loss function 惩罚项；
6. trial pruning；
7. 后处理筛选。

一个很典型的例子是差分对。差分对 M1/M2 不应该分别独立采样，否则优化器可能给出不匹配的尺寸，既不符合电路结构，也会放大搜索空间。

更合理的做法是共享同一组变量：

```python
w_in = trial.suggest_float("w_in", 1e-6, 50e-6, log=True)
l_in = trial.suggest_float("l_in", 180e-9, 1e-6)

params["M1.w"] = w_in
params["M2.w"] = w_in
params["M1.l"] = l_in
params["M2.l"] = l_in
```

这段代码表达的是工程思想：优化器看到的是抽象变量 `w_in` 和 `l_in`，而电路实现层把它们绑定到两个匹配器件上。

类似地，电流镜、互补支路、负载器件、CMFB 支路和补偿网络都可能需要参数绑定或比例约束。如果搜索空间设置得不合理，即使优化算法再先进，也很难得到有意义的结果。

## 9. TPE 方法的局限性与改进方向

TPE 并不是万能工具，它更像是一个高效的参数探索策略。使用时需要清楚它的边界。

首先，TPE 不保证全局最优。它根据历史样本估计分布，如果早期样本覆盖不足，或者搜索空间设计得很差，TPE 也可能集中到不理想区域。

其次，强相关变量如果处理不好，采样效率会下降。例如多个器件宽度、电流分配和补偿参数之间往往高度耦合。单纯让优化器独立采样每个变量，可能会产生大量不合理组合。

第三，多目标优化需要额外设计。TPE 本身可以最小化一个 scalar loss，但模拟电路常常同时关心 gain、bandwidth、phase margin、power、area 和 layout friendliness。如何把这些指标合成一个合理 loss，或者如何做多目标筛选，是另一个需要认真设计的问题。

第四，约束处理主要依赖 search space 和 loss function。TPE 不会天然知道“相位裕度必须大于 60 度”或者“输入对必须匹配”。这些规则必须由工程师写进参数空间、约束惩罚、precheck 或后处理筛选中。

因此，在实际使用中，TPE 通常需要和以下工程手段结合：

- 合理的参数上下界；
- gm-Id 或经验 sizing 给出的初始区域；
- 对称器件参数绑定；
- 失败 trial 的稳定惩罚；
- 分阶段优化策略；
- 可读的 history 和 summary；
- 对最终候选点的重新仿真与人工审查。

优化器不是替代工程师，而是帮助工程师更高效地探索设计空间。电路知识仍然决定搜索空间怎么设、loss 怎么写、结果怎么判断。

## 10. 小结

本文把系列文章推进到了优化算法层。前五篇已经把模拟电路设计变成了一个可调用的黑箱优化问题，而 TPE 解决的是下一步：在仿真代价昂贵、目标函数不可导、变量类型混合、失败 trial 很常见的情况下，如何更聪明地选择下一组候选参数。

TPE 的核心思想可以概括为一句话：

> 从历史 trial 中学习“好参数通常长什么样”，再优先尝试那些像好样本、不像差样本的候选点。

数学上，它通过：

```text
l(x) = p(x | y < y*)
g(x) = p(x | y ≥ y*)
```

把历史样本分成好样本和普通样本，并倾向于选择 `l(x)/g(x)` 较大的候选点。工程上，这意味着 TPE 能在有限 Spectre 仿真预算下，把搜索逐步集中到更有希望满足指标的参数区域。

但需要再次强调：TPE 只能看到 loss。它真正优化的不是 Gain、GBW、PM、SR 本身，而是这些指标经过损失函数和约束处理后形成的数值。因此，优化效果不仅取决于 TPE 算法本身，也强烈依赖 search space、参数绑定、失败处理和 loss function。

下一篇文章将继续讨论：面向模拟 IC 的损失函数设计与约束处理。因为只有把指标、约束和工程偏好合理地写进 loss，TPE 这样的优化器才知道什么叫“更好的电路”。
