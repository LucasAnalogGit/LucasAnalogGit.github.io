# 面向模拟 IC 的损失函数设计与约束处理

## 1. 背景：优化器真正看到的是 loss

上一篇文章介绍了如何使用 TPE 在高维、混合类型的模拟电路参数空间中选择下一组候选参数。但 TPE 并不知道什么是 Gain、GBW、PM、SR 或 Power，它真正看到的只有一串 trial 及其对应的标量目标值。

一次完整评价可以简化为：

```text
x_k → simulation → metrics_k → loss_k
```

其中，`x_k` 是第 `k` 组设计参数，`metrics_k` 是 Spectre/OCEAN 仿真后提取的性能指标，`loss_k` 则是交给优化器的评分。对于最小化问题，优化器只会尝试让这个数字越来越小。

因此，loss function 的本质可以写成：

```text
loss = Score(metrics, constraints)
```

从工程角度看，loss function 更像是优化器的评分规则。优化器并不知道什么是好电路，它只知道哪个 loss 更小。如果评分规则设计错了，优化器会很认真地把电路优化到错误方向。

例如，只奖励 GBW 而不约束相位裕度和功耗时，搜索可能偏向更大的偏置电流和更激进的补偿参数。最终得到的设计速度很快，却可能稳定性不足、功耗过高，甚至没有可接受的 PVT 裕量。问题不在 TPE，而在于我们没有把“好电路”的完整含义写进 loss。

一个实用的自动优化闭环因此应该是：

```text
结构化设计参数
        ↓
Virtuoso / Spectre 仿真
        ↓
Gain、GBW、PM、SR、Power 等 metrics
        ↓
归一化、约束判断与惩罚项
        ↓
scalar loss
        ↓
TPE 选择下一组参数
```

这也是为什么 loss function 往往比优化算法本身更接近模拟设计经验：搜索算法负责“去哪里试”，而 loss 负责定义“什么值得继续寻找”。

## 2. 为什么模拟电路优化不能只用“达标/不达标”

### 2.1 硬阈值判断的问题

最直接的评分方式，是检查每个指标是否满足规格：全部满足记为 0，否则记为 1。这个方法看起来清楚，却丢失了最重要的信息：不达标的程度。

假设两个设计都没有达到 60° 的 PM 要求：

| 设计 | PM | 硬阈值结果 |
|---|---:|---:|
| A | 58° | 不达标 |
| B | 12° | 不达标 |

如果两者得到同一个 loss，优化器就无法判断 A 已经接近可行域，而 B 可能处于完全不同的工作状态。类似地，Gain 差 1 dB 和差 30 dB、GBW 低 5% 和低一个数量级，也不应该得到相同评价。

更合理的做法是使用连续惩罚。以“越大越好”的指标为例：

```text
r_i = max(0, (target_i - value_i) / target_i)
p_i = r_i^2
```

当指标达标时，`p_i = 0`；未达标时，惩罚会随着相对缺口增大。平方项让严重偏离规格的设计受到更强惩罚，同时保留“还差多少”的排序信息。

在模拟 IC 优化中，这个连续性非常重要。一个好的 loss 不只是惩罚失败，也要告诉优化器应该往哪里改。

### 2.2 指标之间的 trade-off

模拟电路性能通常不会沿同一个方向一起改善。几个常见场景是：

- Gain 已经满足，但 GBW 不够；
- PM 很高，但补偿电容把带宽拖低；
- SR 达标，但偏置电流导致功耗过大；
- GBW 远高于目标，却没有足够的稳定性裕量；
- 所有电气指标都达标，但 MOS 尺寸极端，难以版图实现。

因此，loss 不能只奖励某一个“明星指标”。一个常见的单目标表达是加权惩罚和：

```text
L(x) = Σ w_i p_i(x)
```

其中，`p_i(x)` 是第 `i` 个指标的归一化惩罚，`w_i` 是其权重。高权重指标会更强地影响搜索方向，低权重指标则更容易被其他目标让步。

这里的权重不是纯数学参数，而是设计优先级的表达。如果 PM 经常不达标，可以提高稳定性相关惩罚的权重；如果设计已经稳定但功耗偏高，则可以在可行域内增加 power penalty。副作用也同样直接：某个权重过大时，优化器可能为它牺牲其他指标；权重过小时，关键规格可能长期得不到改善。

### 2.3 仿真失败与异常指标

自动优化中的失败 trial 很常见，包括：

- netlist 生成失败；
- Spectre 不收敛或超时；
- DC operating point 不存在；
- 结果文件缺失；
- PM 或交越频率无法计算；
- 波形中出现 `NaN`、`inf` 或明显无效的数据。

这些情况不能让整个优化程序崩溃，也不能被静默忽略。最基本的处理是：

```text
if simulation_failed:
    return large_penalty
```

大惩罚把无效设计排在正常设计之后，但还需要记录 trial 参数、失败阶段、错误类型和运行时间。失败比例本身也是诊断信号：如果大量候选点都在 DC 或 netlist 阶段失败，通常说明 search space、参数映射或 testbench 存在问题，而不只是“优化器还不够聪明”。

### 2.4 loss function 是工程经验的入口

模拟设计经验不会因为引入优化器而消失，它只是换了一种表达方式：

- 参数上下界表达工艺和经验范围；
- 参数绑定表达差分对、电流镜等结构关系；
- 指标提取规则表达测量口径；
- 约束表达必须满足的设计条件；
- 权重和惩罚表达性能优先级；
- 过设计惩罚表达“够用且均衡”的工程偏好。

这意味着 loss function 不应被当作优化流程末尾的一段简单加法。它是工程师经验进入优化器的重要入口，也是自动化结果是否有实际意义的关键边界。

## 3. 从性能指标到优化目标

### 3.1 metrics 向量

对一组设计参数 `x`，仿真与指标提取模块可以返回一个指标向量：

```text
m(x) = [Gain(x), GBW(x), PM(x), SR(x), Power(x), Area(x)]
```

其中，`m(x)` 仍然是多维信息，TPE 的单目标优化接口通常需要一个标量。loss function 的任务就是完成映射：

```text
L(x) = Φ(m(x))
```

这个标量是优化器比较不同 trial 的评分，通常越小越好。但 `Φ` 不能只是把原始数值直接相加，因为这些指标的方向、单位和量级都不同。

工程上最好让 metric extraction 与 loss function 解耦。前者只负责可靠地回答“仿真测到了什么”，后者负责回答“这些结果是否满足当前设计目标”。这样更换规格、权重或优化算法时，不必改动波形读取逻辑；同一套 metrics 也可以被不同的评价策略复用。

### 3.2 指标归一化

Gain 的单位是 dB，GBW 可能以 Hz 表示，PM 是 degree，SR 是 V/µs，Power 是 W。直接把这些原始数值相加，结果通常没有物理意义。例如一个以 `10^8` 表示的 GBW 数值，会轻易淹没以几十表示的 PM。

对“越大越好”的指标，可以先计算相对缺口：

```text
r_i = (target_i - value_i) / target_i
p_i = max(0, r_i)^2
```

归一化的目的不是让公式更漂亮，而是避免某个数值量级很大的指标在 loss 中占据不合理的主导地位。经过归一化后，“Gain 低于目标 10%”和“GBW 低于目标 10%”至少进入了可比较的尺度，再由权重表达二者的优先级。

实际工程中，归一化尺度不一定必须等于 target。当目标接近零、指标可正可负，或者指标跨越多个数量级时，可以使用独立的工程尺度 `s_i`、对数比值或经过验证的统计尺度。归一化方式本身也会改变搜索地形，需要和指标物理意义一起设计。

### 3.3 越大越好、越小越好与目标区间型指标

不同指标需要不同的惩罚方向。

**越大越好**的规格常见于 Gain、GBW、PM 和 SR。最低目标惩罚可以写为：

```text
p_i = max(0, (target_i - value_i) / target_i)^2
```

它解决的是“指标不足多少”的问题。指标一旦达到最低目标，这一项就不再继续奖励，避免优化器仅为了抬高一个已经达标的指标而消耗资源。

**越小越好**的规格常见于 Power、Area、Noise 或 settling time：

```text
p_i = max(0, (value_i - limit_i) / limit_i)^2
```

它只惩罚超过上限的部分。若 Power 是可行域内还需要持续压低的目标，也可以把它作为单独的效率项，而不只是在超限时处罚。

**目标区间型**指标常见于输出共模、偏置电压和某些工作点电压：

```text
p_i = 0,                                  lower_i ≤ value_i ≤ upper_i
p_i = ((lower_i - value_i) / s_i)^2,      value_i < lower_i
p_i = ((value_i - upper_i) / s_i)^2,      value_i > upper_i
```

这个表达解决的是“保持在窗口内”，而不是一味追求更大或更小。例如输出共模过高和过低都可能压缩摆幅或破坏器件工作区，区间惩罚能给出正确的双向引导。

需要注意，PM 虽然通常先按“不得低于下限”处理，却不意味着越高越好。极高 PM 可能只是过度补偿的结果，真正需要约束的往往是 PM、GBW、功耗和补偿元件之间的组合关系。

### 3.4 单目标 loss 与多目标优化

将所有惩罚合成一个标量，优点是接口简单，适合直接交给 TPE：

```text
L(x) = w_gain p_gain + w_gbw p_gbw + w_pm p_pm
     + w_sr p_sr + w_power p_power + w_area p_area
```

这个公式解决的是“如何给候选设计排一个总顺序”。它让优化器可以明确比较任意两个 trial，但也会把部分 trade-off 隐藏在权重中。不同权重可能得到不同的最优解，而且加权和不一定能充分表达复杂的非凸 Pareto 前沿。

如果多个目标之间没有稳定的优先级，例如功耗与速度都需要保留多种候选方案，可以采用多目标优化并输出 Pareto front，再由工程师结合系统需求筛选。即使如此，工艺合法性、仿真有效性和最低性能规格通常仍应作为约束独立处理，而不是全部混入 Pareto 目标。

## 4. 常见指标的惩罚项设计

### 4.1 Gain 不足的惩罚

Gain 不足通常意味着开环增益没有达到系统精度要求。最常见的做法是只惩罚低于最低规格的部分：

```text
p_gain = w_gain · max(0, (Gain_min - Gain) / s_gain)^2
```

`s_gain` 可以取目标值，也可以取一个有明确工程意义的 dB 尺度。平方惩罚会让严重缺口快速抬高 loss，同时允许接近目标的设计得到更好排序。

它的副作用是：如果 `w_gain` 太大，优化器可能偏向更长沟道、更大输出电阻或更重负载，从而牺牲速度与面积。因此 Gain 达标后是否继续奖励，应由整体目标决定，而不是默认“越高越好”。

### 4.2 GBW 不足的惩罚

GBW 常跨越较大的数值范围，既可以使用相对缺口，也可以使用对数尺度：

```text
p_gbw = w_gbw · max(0, log(GBW_target / GBW))^2
```

对数形式表达的是“相差多少倍”，适合搜索空间跨度较大的场景；相对缺口则更直观，适合已经靠近目标的局部优化。两种形式都会影响优化器对远离目标样本的敏感程度，不能只因为公式简洁就随意替换。

还要防止 GBW 成为单一主导目标。只追求带宽，可能导致补偿过激、偏置电流增加或 PM 下降，所以 GBW 通常要和稳定性及功耗一起评价。

### 4.3 PM 不足与过度补偿

PM 低于下限时，应使用连续的稳定性惩罚：

```text
p_pm_low = w_pm · max(0, (PM_min - PM) / s_pm)^2
```

但“PM 很高”不一定应该直接受罚。问题通常不是这个角度本身，而是为了获得高 PM 付出的带宽、功耗或补偿面积代价。如果设计目标已经给出了偏好的 PM 区间，也可以增加温和的上侧惩罚；否则更稳妥的方式是让 GBW、Power 和补偿元件成本共同约束过度补偿。

此外，PM 的提取必须可靠。没有明确环路定义、交越点不唯一或幅频曲线异常时，得到的数值可能没有可比性。这类问题应该先被 metric validity 捕获，而不是让一个看似正常的 PM 数字进入 loss。

### 4.4 SR 不足的惩罚

SR 反映大信号充放电能力。对差分或全差分电路，正向和负向转换速度都需要检查：

```text
SR_effective = min(SR_positive, SR_negative)
p_sr = w_sr · max(0, (SR_min - SR_effective) / s_sr)^2
```

取较差方向作为有效 SR，可以防止优化器用一侧的优异结果掩盖另一侧的不足。SR 远高于目标时也要结合功耗判断，因为它往往意味着更大的充放电电流。

SR 还依赖输入阶跃、负载、测量窗口和斜率提取方法。测试条件变化后，目标和归一化尺度也应同步调整。

### 4.5 Power 过大的惩罚

当功耗有明确上限时，可以使用：

```text
p_power = w_power · max(0, (Power - Power_max) / Power_max)^2
```

这能把超限设计推向可行域，但对所有低于上限的设计都给零分，无法继续区分“刚好达标”和“更高效”。因此常见做法是先把 `Power ≤ Power_max` 作为约束，找到可行解后，再在可行域内部把归一化功耗作为需要持续最小化的目标。

这种分层能避免优化早期为了省电而选择根本不能工作的弱偏置点，也能在性能达标后继续压缩功耗。

### 4.6 面积与器件尺寸惩罚

原理图优化阶段通常还没有最终版图面积，但可以用总栅面积、总宽度、最大 W/L、器件倍数、补偿电容等作为可实现性代理：

```text
Area_proxy = Σ W_i L_i + k_c C_c + k_r R_c
```

这个表达用于比较候选设计的相对实现成本，不应被误解为最终版图面积。真实版图还受匹配、走线、保护环、器件折叠、dummy、CMFB 结构和设计规则影响。

尺寸惩罚可以阻止优化器生成“仿真上达标、版图上难以接受”的极端方案，但代理模型过于粗糙时，也可能错误排除本来可以通过合理布局实现的设计。因此，面积代理应保持可解释，并在后续版图阶段重新校准。

## 5. 硬约束与软约束

直观地说，硬约束决定一个设计“有没有资格进入候选集”，软约束则表达“虽然还能接受，但更希望它往哪个方向改善”。两者都可以使用连续惩罚，区别在于是否改变可行性判断。

数学上，若硬约束写成 `c_j(x) ≤ 0`，则可行性条件为：

```text
feasible(x) = True，当且仅当所有 c_j(x) ≤ 0
```

软约束即使被违反，也不会把 `feasible` 改为 `False`，而是把对应 penalty 加入 loss。在模拟 IC 优化中，这意味着“没有 DC 工作点”可以作为硬失败，“器件工作点离偏好区间稍远”则可以作为软引导。硬约束负责守住物理和规格底线，软约束负责在仍可工作的设计之间表达工程偏好。

### 5.1 工艺约束

硬约束对应不能接受或没有物理意义的设计，例如器件尺寸超出 PDK 允许范围、整数参数非法、关键器件缺失或参数映射失败。它们最好在仿真前通过 search space 边界和 precheck 阻止，而不是每次都送入 Spectre 后再处罚。

可以把硬失败写成独立的高损失等级：

```text
L_invalid = B_invalid + P_error
```

`B_invalid` 是明显高于正常可行解的基准值，`P_error` 可以区分不同失败程度或类型。这样既保证非法设计不会胜出，也避免向优化器返回 `NaN` 或无穷值。

### 5.2 电路工作区约束

MOS 是否处于期望工作区、输出是否饱和、关键节点是否保留足够余量，既可以是硬约束，也可以是软约束，取决于严重程度。

直观上，完全失去 DC 工作点属于硬失败；某个器件离饱和边界只差少量裕量，则可以用连续惩罚引导。区间型工作点约束可以写成：

```text
p_op = w_op · distance(value, valid_interval)^2
```

它解决的是“离可接受工作区还有多远”。如果所有工作区违规都得到同一个大 loss，优化器无法知道哪个候选点更容易通过调整偏置回到正常区域。

### 5.3 版图可实现性约束

器件尺寸虽然在 PDK 数值范围内，也可能不适合实际版图。例如单管宽度过大、finger 数不合理、匹配器件几何不一致、补偿电容面积过高，或者关键支路不具备对称实现条件。

这类规则通常适合分两层处理：明显违反制造或结构规则的候选点作为硬约束；影响布局质量但仍可实现的候选点作为软惩罚。软约束不会直接把设计判为不可行，但会推动搜索远离不利于布局和匹配的区域。

需要强调的是，原理图阶段的 layout-friendly loss 只是代理。最终候选点仍需经过版图、寄生提取和 post-layout 验证。

### 5.4 failed trial 的处理

对优化器来说，一个不满足 PM 的设计和一个完全无法仿真的设计都需要被惩罚，但二者不应完全等价。前者可能只差一点，后者则应该被快速排除。

一种实用的两层评分方式是：

```text
不可仿真或数据无效：L = B_failed
可仿真但不满足约束：L = B_infeasible + Σ w_i p_i
满足全部硬约束：    L = L_efficiency
```

其中应满足：

```text
B_failed > B_infeasible > 可行设计的典型 loss 范围
```

这套分层解决了两个工程问题：首先，无效结果不会意外成为最优值；其次，可仿真但尚未达标的设计仍可按违反程度排序，从而引导优化器逐步靠近可行域。

每个 failed trial 至少应记录状态、失败阶段、错误摘要、参数和耗时。若失败比例持续过高，应优先检查 search space、testbench、netlist 和指标提取，而不是简单把惩罚值继续调大。

## 6. 可行域与不可行域

### 6.1 可行设计的定义

假设最低规格为 Gain、GBW、PM、SR 不低于目标，Power 不高于上限，那么可行域可写为：

```text
Ω_feasible = {x ∈ Ω |
              Gain(x)  ≥ Gain_min,
              GBW(x)   ≥ GBW_min,
              PM(x)    ≥ PM_min,
              SR(x)    ≥ SR_min,
              Power(x) ≤ Power_max}
```

这个集合表达的是“所有硬规格同时满足”的参数区域。它通常不是规则矩形，而是由电路非线性、器件工作区和多个 trade-off 共同形成的复杂区域。优化早期，大多数 trial 位于可行域之外是正常现象。

### 6.2 不可行设计是否完全丢弃

不可仿真设计通常只能提供有限信息，但可仿真且接近规格的不可行设计非常有价值。如果把所有不可行点都丢弃，优化器会失去可行域边界附近的方向信息。

更好的做法是保留这些 trial，并使用归一化违反量排序：

```text
V(x) = Σ w_i · violation_i(x)
```

`V(x)` 越小，说明候选点越接近同时满足约束。它不能保证下一次修改一定进入可行域，但能让采样器区分“接近成功”和“完全偏离”的区域。

### 6.3 从不可行区域引导到可行区域

自动优化的推进过程可以理解为：

```text
不可仿真区域
        ↓
可仿真但不达标区域
        ↓
刚好满足指标的可行区域
        ↓
具有更好功耗、面积和鲁棒性的优选区域
```

连续 penalty 的作用，就是为前两个箭头提供方向。比如一个设计 Gain 已经满足、GBW 只差 10%、PM 只差 3°，它应该明显优于一个所有指标都严重偏离的设计。优化器不理解这些物理含义，但可以通过 loss 的相对大小学到哪些参数区域更接近可行域。

### 6.4 可行解内部的二次优化

所有达标设计并不等价。进入可行域后，工程上通常还要比较：

- 功耗与面积；
- 关键指标的适度裕量；
- 器件工作区和偏置鲁棒性；
- PVT 与 Monte Carlo 表现；
- 版图可实现性；
- 对参数扰动的敏感度。

因此可以采用分阶段 loss：

```text
Stage 1: validity loss       先找到可仿真、可工作的设计
Stage 2: specification loss 推动 Gain、GBW、PM、SR 达标
Stage 3: efficiency loss    在可行域内优化 Power、Area 和裕量
```

分阶段的好处是让优化器先解决“能不能工作”，再解决“性能是否达标”，最后再优化“是否优雅”。但阶段切换条件必须明确，并保留统一的最终评价规则，否则不同阶段的 loss 数值可能无法直接比较。

## 7. 过设计惩罚：为什么不是指标越高越好

在模拟电路中，超过规格并不自动等于更好的系统设计：

- GBW 远高于目标，可能来自过大的偏置电流或过小的补偿；
- PM 很高，可能是以明显牺牲带宽为代价；
- Gain 远高于目标，可能依赖更长沟道或更大负载；
- SR 远高于目标，可能伴随不必要的功耗；
- 极端器件尺寸可能让版图、匹配和寄生变得更困难。

如果某个指标存在偏好的上限，可以引入 over-design penalty：

```text
p_over = w_over · max(0, (value_i - upper_preferred_i) / upper_preferred_i)^2
```

这个惩罚解决的是“超过多少已经不再值得”。它不是为了阻止性能提升，而是避免优化器为了一个指标过度牺牲其他工程目标。

实际使用时，最好给目标以上留出一段免费裕量：

```text
p_over = w_over · max(0, normalized_excess - free_margin)^2
```

`free_margin` 允许设计保留合理余量，超过后才开始惩罚。还可以对惩罚设置上限，防止过设计项反过来淹没真正的硬约束。

过设计惩罚通常只应在设计已经可行时启用。否则，一个 GBW 不足但功耗很低的不可行设计，可能因为“效率好”而被错误评价。更稳妥的优先级是：先满足硬规格，再在可行解之间处理过设计和效率。

## 8. 一个简化的 loss function 示例

下面的 Python 代码只用于说明接口和评分思想，不对应任何具体工程实现：

```python
FAILED_LOSS = 1e9


def larger_is_better(value, target, weight=1.0):
    if value is None:
        return weight * 1e6
    ratio = max(0.0, (target - value) / target)
    return weight * ratio * ratio


def smaller_is_better(value, limit, weight=1.0):
    if value is None:
        return weight * 1e6
    ratio = max(0.0, (value - limit) / limit)
    return weight * ratio * ratio


def interval_penalty(value, lower, upper, scale, weight=1.0):
    if value is None:
        return weight * 1e6
    if value < lower:
        distance = (lower - value) / scale
    elif value > upper:
        distance = (value - upper) / scale
    else:
        distance = 0.0
    return weight * distance * distance


def compute_loss(metrics):
    if not metrics.get("valid", False):
        return FAILED_LOSS

    loss = 0.0
    loss += larger_is_better(metrics.get("gain_db"), 80.0, weight=2.0)
    loss += larger_is_better(metrics.get("gbw_hz"), 50e6, weight=1.5)
    loss += larger_is_better(metrics.get("pm_deg"), 60.0, weight=2.0)
    loss += larger_is_better(metrics.get("sr_v_us"), 100.0, weight=1.0)
    loss += smaller_is_better(metrics.get("power_w"), 2e-3, weight=1.0)

    # 示例：把输出共模限制在一个可接受窗口内。
    loss += interval_penalty(
        metrics.get("output_cm_v"),
        lower=0.8,
        upper=1.0,
        scale=0.1,
        weight=1.5,
    )
    return loss
```

这段示例表达了几条基本原则：缺失指标不能产生 `NaN`；不同方向的指标使用不同惩罚；所有项先归一化，再通过权重合成；仿真无效时返回有限的大惩罚。

它不是完整工程实现。真实流程还需要处理单位转换、正负 SR、差分增益、多个 corner、器件工作区、输出摆幅、CMFB、稳定性测量、尺寸合法性、错误分类和 loss breakdown。示例中的目标与权重也只是演示值，不能直接作为某个实际电路的规格。

工程实现中，建议让 `compute_loss(metrics, objective_spec)` 只依赖结构化 metrics 和声明式目标配置。仿真执行、波形读取、指标提取以及优化算法都不应被写进同一个函数。这样既便于单元测试，也能在不改动 TPE 流程的情况下调整评分策略。

## 9. 工程实践中的注意事项

一个能运行的 loss 不一定是一个好 loss。实际使用时，下面这些问题比公式形式更值得持续检查：

1. **不要只用单一指标判断设计好坏。** Gain、GBW、PM、SR、Power 和实现成本之间存在真实 trade-off。
2. **不要把所有不达标设计都给同一个 loss。** 对可仿真的候选点保留连续违反量，优化器才知道谁更接近成功。
3. **先确认单位，再做归一化。** Hz、MHz，W、mW，V/s、V/µs 混用时，loss 可以在不报错的情况下完全失真。
4. **权重必须反映设计优先级。** 每次调整权重都应观察 loss breakdown，而不是只看总 loss 是否下降。
5. **仿真失败必须记录。** 失败阶段、错误类别和原始参数有助于收缩 search space 和修复 testbench。
6. **failed trial 过多时先检查流程。** 高失败率可能来自边界过宽、参数映射错误、初始条件不合理或指标提取不稳。
7. **差分电路应使用差分指标。** 单端增益、共模结果和差分结果不能混为一谈，正负 SR 也应分别测量。
8. **PM、GBW 的提取口径必须可靠。** 环路断点、注入方法、交越点选择和相位展开方式应固定并验证。
9. **loss function 不应与 TPE 强耦合。** 同一个评价函数应能被随机搜索、TPE 或其他优化器复用。
10. **不同阶段可以采用不同策略。** 早期关注有效性，中期关注规格，后期关注功耗、面积和鲁棒性。
11. **过设计不一定是好设计。** 对超过规格的收益设置合理边界，避免优化器追逐没有系统价值的数字。
12. **达标只是第一步。** 最终候选点还需要在 PVT、Monte Carlo、版图寄生和 post-layout 条件下重新验证。

此外，建议每次 trial 同时保存总 loss 和各项 breakdown，例如约束违反量、软惩罚、效率目标与过设计项。只有这样，才能回答“loss 为什么下降”“哪个指标主导了搜索”“最优点是否只是利用了评分漏洞”等关键问题。

权重调整也不应只凭一次最优结果。更稳妥的方式是观察一批 trial 的分布：哪些约束长期无法满足，哪些项几乎总是为零，哪些惩罚持续占据绝对主导。如果某项长期独占 loss，可能需要修改归一化尺度、搜索空间或测试条件，而不仅是机械地降低权重。

## 10. 小结

本文补上了 TPE 与实际电路指标之间最关键的一层：如何把 Gain、GBW、PM、SR、Power、Area 以及工作区和实现约束，转换为优化器可以比较的标量 loss。

核心思路可以概括为：

```text
metrics
        ↓
normalization
        ↓
constraint penalty
        ↓
loss function
        ↓
optimizer objective
```

一个好的 loss 需要同时完成几件事：无效设计要被稳定排除；未达标设计要按距离排序；不同指标要在可比较尺度上加权；可行设计内部还要继续比较功耗、面积、裕量和版图可实现性；对于没有系统价值的性能冗余，则需要谨慎处理过设计。

TPE 负责选择下一组候选参数，但它不能直接理解模拟电路指标。真正告诉优化器“什么是好电路”的，是 loss function。loss function 的设计质量，往往决定了自动优化流程能否收敛到工程上有意义的结果。

到这里，这个系列已经具备一条完整闭环：生成电路、运行仿真、提取指标、计算 loss，再由 TPE 选择下一组参数。下一篇将进入具体电路案例，例如“五管 OTA 自动优化案例：从原理图生成到性能收敛”，观察这套方法如何在真实的参数迭代中工作，以及如何判断所谓的“最优结果”是否真的可用。
