# 将模拟电路设计转化为黑箱优化问题

## 1. 背景：从自动化流程到优化建模

前四篇文章依次搭建了模拟 IC 自动化设计的执行链路：用 virtuoso-bridge 打通 Python 与 Cadence Virtuoso，用 SKILL 自动生成原理图，用 Spectre/OCEAN 构建可重复的自动仿真流程，再用 Python 从仿真结果中提取结构化性能指标。到这里，自动化系统已经能够根据一组设计参数生成电路、运行仿真，并输出带有效性标记的 metrics。

但执行链路再顺畅，也还只是“把工程师原本手动做的事自动化了一遍”。真正的自动设计，需要让算法替代经验去决定“下一组参数取什么”。要做到这一点，必须先跨过一道建模的门槛：

> 自动化仿真并不等价于自动化设计。要让优化算法参与模拟电路设计，首先必须把传统依赖经验和手工调参的电路设计过程，转化为可被计算机搜索、评价和迭代的数学问题。

本文就是这道门槛。它把系列文章从“自动化执行流程”推进到“优化建模”，把电路设计重新表述为一个高维、有约束、评价代价昂贵的搜索问题：

```text
电路拓扑 + 设计变量 + 工艺约束
        ↓
多维参数空间
        ↓
Spectre 黑箱仿真函数
        ↓
性能指标
        ↓
优化问题
```

前四篇解决的是“怎么算一次”，本文解决的是“怎么把设计写成一个能反复求解的数学对象”。只有完成这一步，后续的优化算法和损失函数才有明确的作用对象。

## 2. 为什么模拟电路设计可以看作优化问题

手动模拟设计的日常，其实非常接近一个搜索过程：工程师根据经验设定一组器件尺寸和偏置，跑仿真，看 gain、GBW、PM、SR、power 是否达标，不达标就回头改参数，再跑一次。这个"改参数—仿真—看指标—再改"的循环，本质上就是在一个参数空间里做迭代搜索。

把这个过程形式化，需要三个要素：

1. **一个可调的输入**：器件宽长、倍数、偏置电流、补偿网络等设计变量，构成参数向量 `x`；
2. **一个可评价的输出**：仿真提取出的性能指标向量 `m`，以及是否满足约束；
3. **一个可比较的标准**：把指标映射成一个可最小化的标量，即损失函数 `L(x)`。

一旦这三者齐备，"设计一个电路"就等价于"在参数空间中寻找使损失最小、且满足约束的点"。经验并没有消失，而是被编码进了参数空间的边界、变量的绑定关系和损失函数的目标里。区别在于：搜索这件事，可以交给算法去做，而且做得比人更系统、更可复现、更能覆盖高维空间。

## 3. 电路拓扑的数学抽象

要把设计写成优化问题，第一步是把电路本身表示成程序能操作的数据结构。

### 3.1 从原理图到图结构

一张模拟电路原理图，可以抽象为一个图：

```text
G = (V, E)
```

其中：

- `V` 是节点（net）集合，例如 `vinp`、`vinn`、`voutp`、`voutn`、`vdd`、`vss`、`bias`、`tail`；
- `E` 是器件或连接关系，例如 MOS、电阻、电容、电流源、受控源；
- 每个器件可以看作一条带属性的边，或者更准确地说，一条连接多个 net 的超边。

需要特别注意 MOS 管不是简单的二端器件。一个 MOS 至少有 `D/G/S/B` 四个 terminal，因此它不能用普通图里"一条边连两个点"来表达，而要看作一个把多个 net 绑在一起的多端元件。

这里有一个容易忽略但很关键的观念：

> 对模拟电路而言，图结构的核心不是几何位置，而是电气连接关系。只要器件 terminal 与 net 的映射关系明确，电路拓扑就可以被程序化描述。

也就是说，我们并不关心某个 MOS 画在原理图的左上还是右下，只关心它的 D/G/S/B 分别接到了哪个 net。版图和几何是后话，拓扑层面只需要连接关系。

### 3.2 器件、节点与连接关系

于是拓扑可以落到一个很朴素的数据结构上：一个器件列表，每个器件带类型和 terminal→net 的映射；再加上一个 net 列表。

```python
# 仅为示意：以差分对为例的拓扑描述
topology = {
    "devices": [
        {"name": "M1", "type": "nmos",
         "nodes": {"D": "voutn", "G": "vinp", "S": "tail", "B": "vss"}},
        {"name": "M2", "type": "nmos",
         "nodes": {"D": "voutp", "G": "vinn", "S": "tail", "B": "vss"}},
    ],
    "nets": ["vinp", "vinn", "voutp", "voutn", "tail", "vdd", "vss"],
}
```

这段结构只描述"谁连到谁"，不含任何尺寸。它足以唯一确定一个拓扑，也足以让程序生成 netlist、检查连接完整性、识别对称结构。

### 3.3 邻接矩阵与关联矩阵

图结构可以进一步矩阵化，便于用线性代数或图算法处理。常用的有三类矩阵，它们回答不同的问题。

**邻接矩阵**描述节点之间是否被器件连接：

```text
A_{ij} = 1, 若节点 i 与节点 j 之间存在器件连接
         0, 否则
```

邻接矩阵适合表达整体连通性，但对 MOS 这类多端器件表达能力有限——它只能说"这两个 net 之间有东西相连"，却说不清是漏源还是栅衬。

**关联矩阵**描述器件端口与 net 之间的连接关系，表达能力更强：

```text
B_{k,n} = 1, 若器件端口 k 连接到节点 n
          0, 否则
```

对多端器件，更清晰的写法是带 terminal 维度的三维索引：

```text
B_{d,t,n}
```

其中 `d` 是第 d 个器件，`t` 是该器件的 terminal（例如 D/G/S/B），`n` 是电路 net。这种形式能干净地区分 MOS、电容、电阻、电流源等不同器件的每个端口具体接到哪个 net，是拓扑到 netlist 之间最自然的桥梁。

### 3.4 拓扑固定与参数可变

矩阵化之后，拓扑和尺寸就自然分开了。器件参数单独放进一个参数矩阵：

```text
P = [
  [W_1, L_1, m_1, nf_1],
  [W_2, L_2, m_2, nf_2],
  ...
]
```

对照来看：

- **拓扑矩阵（邻接/关联）描述"谁和谁连接"**；
- **参数矩阵描述"每个器件取什么尺寸"**。

这对自动优化至关重要：在一个固定拓扑的优化问题中，连接关系（关联矩阵）保持不变，优化器只在参数矩阵 `P` 上搜索。这样既避免了优化器生成无法仿真或无意义的连接结构，又把问题收敛成一个规模可控的数值搜索。

## 4. 从电路拓扑到多维参数空间

拓扑固定后，剩下的自由度就是参数。把这些自由度收集起来，就得到优化器要搜索的多维参数空间。

### 4.1 设计变量的定义

设计变量就是电路中允许被优化器调整的量：器件宽长 `W/L`、倍数 `m`、finger 数 `nf`、补偿电容 `C_c`、调零电阻 `R_z`、偏置电流 `I_bias` 等。把它们串成一个向量：

```text
x = [W_1, L_1, m_1, W_2, L_2, m_2, ..., C_c, R_z, I_bias]
```

`x` 的每一维对应一个可调旋钮，整个向量就是一个候选设计点。

### 4.2 连续变量、离散变量与类别变量

设计变量并非都是连续的，工程上通常要区分三类：

| 变量类型 | 示例 | 特点 |
|---|---|---|
| 连续变量 | W、L、I_bias、C_c | 在给定范围内连续取值 |
| 离散变量 | multiplier、finger number | 通常为正整数 |
| 类别变量 | 器件类型、补偿结构选项 | 只能从有限候选集合中选择 |

区分变量类型不是形式主义。优化器对不同类型的变量要用不同的分布去建模和采样：连续变量常在对数或线性尺度上采样，离散变量要落在整数格点上，类别变量则要在候选集合里选择。类型标错，优化器就会在无意义的取值上浪费大量仿真预算。

### 4.3 参数向量 x 的构造

把所有变量合起来，搜索空间是各维取值范围的乘积：

```text
x ∈ Ω ⊂ R^n
```

当包含离散和类别变量时，空间是混合的：

```text
Ω = Ω_continuous × Ω_discrete × Ω_categorical
```

在工程实现里，与其把 `x` 写成一个裸数组，不如用带元信息的声明式结构描述每一维，让优化器知道它的类型、范围和采样尺度：

```python
# 仅为示意：搜索空间声明
search_space = {
    "w_in": {"type": "float", "low": 1e-6,   "high": 50e-6, "log": True},
    "l_in": {"type": "float", "low": 180e-9, "high": 1e-6,  "log": False},
    "m_in": {"type": "int",   "low": 1,      "high": 16},
    "cc":   {"type": "float", "low": 100e-15,"high": 5e-12, "log": True},
}
```

这样，`topology` 描述连接关系，`parameters` 描述某个具体设计点的尺寸，`search_space` 描述优化器被允许探索的范围——三者职责分明。真实工程里，这一层之外还需要 PDK 映射、testbench、仿真配置和指标提取模块配合，但从建模角度看，参数空间的核心就是这张声明表。

### 4.4 参数边界与工艺约束

参数空间不能随意设置，它必须落在工艺和工程允许的范围内。首先是逐变量的边界与整数性：

```text
W_min ≤ W_i ≤ W_max
L_min ≤ L_i ≤ L_max
m_i  ∈ Z^+
nf_i ∈ Z^+
```

其次是结构性约束。差分对要求严格匹配：

```text
W_{M1} = W_{M2},  L_{M1} = L_{M2},  m_{M1} = m_{M2}
```

电流镜可能要求固定的尺寸比例：

```text
(W_{M3}/L_{M3}) / (W_{M4}/L_{M4}) = r
```

补偿网络则要求物理上成立：

```text
C_c > 0,  R_z > 0
```

这些约束不是可有可无的修饰。恰恰相反：

> 参数空间设计得是否合理，直接决定优化器能否找到物理合理、版图可实现、仿真稳定的解。

一个把差分对拆成两个独立变量的搜索空间，几乎注定让优化器把大量预算浪费在失配、无法版图实现的点上。好的参数空间，本身就编码了设计者的结构性经验。

## 5. Spectre 仿真作为黑箱函数

有了参数空间，还需要一个把参数变成性能的映射。这个映射由前四篇搭好的整条链路承担，从优化视角看，它是一个黑箱函数。

### 5.1 从参数到 netlist

给定一个设计点 `x`，第一步是把它写回电路。由于拓扑固定，这一步不改变连接关系，只把 `x` 中的尺寸填进参数化 netlist 模板或 schematic 的 design variables：

```text
x → schematic(x) / netlist(x)
```

工程上常见的做法是维护一个参数化的 netlist 模板（DUT + testbench 连接关系固定，尺寸用占位符表示），每轮用 `x` 渲染出一份具体 netlist。这样拓扑与 sizing 天然解耦。

### 5.2 从 netlist 到仿真结果

第二步把 netlist 交给 Spectre，运行 dc/ac/tran/stb 等 analysis，得到波形、工作点和日志：

```text
netlist(x) → Spectre → waveforms
```

这一步的代价是整个流程里最昂贵的：一次函数评价可能意味着一次或多次晶体管级仿真，耗时从秒到分钟不等。

### 5.3 从仿真结果到 metrics

第三步用第四篇讨论的提取层，把波形转成结构化指标，并判断有效性：

```text
waveforms → metrics = [Gain, GBW, PM, SR, Power, ...]
```

### 5.4 黑箱函数 f(x) 的定义

把三步串起来，整条链路就是一个函数：

```text
m = F(x)
```

其中 `F(·)` 由 schematic 更新、netlist 渲染、Spectre 仿真和指标提取共同构成，`m` 是性能指标向量：

```text
m = [Gain(x), GBW(x), PM(x), SR(x), Power(x)]
```

之所以称它为"黑箱"，是因为它具备下面这些性质：

- 无法写出精确的解析表达式；
- 对参数强非线性；
- 依赖复杂的器件模型；
- PDK 模型不可完全解析；
- 每次函数评价都需要一次或多次仿真，代价高；
- 结果可能出现仿真失败、收敛失败或异常指标。

对优化器来说，`F` 内部发生了什么并不重要——它只能"输入 `x`、得到 `m`（或一个失败标记）"。这正是黑箱优化的典型设定，也决定了后续算法必须"省着花"每一次仿真。

## 6. 性能指标与约束条件

黑箱输出的是一组指标，但优化需要的是"目标 + 约束"。这一节把指标翻译成优化问题的语言。

### 6.1 Gain、GBW、PM、SR、Power

常见的模拟指标各自扮演不同角色。有的是"越大越好"（gain、GBW、SR），有的是"不能低于某个下限"（PM），有的是"越小越好或不能超过上限"（power、面积）。把它们统一写成带方向的目标和阈值，是建模的前提。

### 6.2 显式约束与隐式约束

约束分两类。**显式约束**来自规格书和工艺规则，能直接写成不等式：gain ≥ 目标、PM ≥ 相位裕度下限、power ≤ 功耗上限、`W_min ≤ W ≤ W_max` 等。**隐式约束**则来自"能不能算出来"：DC 工作点是否收敛、MOS 是否落在期望工作区、仿真是否成功、指标是否为有限值。隐式约束往往不写在规格书里，却同样决定一个设计点是否可用。

### 6.3 可行域与不可行设计

把约束收进来，就定义了可行域：

```text
Ω_feasible = { x ∈ Ω | g_i(x) ≤ 0, i = 1,2,...,K }
```

参数空间因此天然分成三层：

```text
参数空间 Ω
 ├── 不可仿真区域        (仿真失败 / 不收敛 / 指标缺失)
 ├── 可仿真但不满足指标区域 (能算出数，但不达标)
 └── 满足所有指标的可行区域 (Ω_feasible)
```

几个工程判断需要落实到代码：不满足约束的设计是不可行解；仿真失败同样视作不可行解；指标为 NaN、inf 或缺失时也要标记为无效。优化器的任务，就是从不可仿真或不达标的区域，一步步搜索到最内层的可行区域，并在其中优化。

### 6.4 多目标权衡

模拟设计几乎总是多目标的：更高的 gain 往往意味着更大的 power，更宽的带宽可能牺牲相位裕度。这些目标彼此冲突，不存在同时最优的单点。处理方式通常有两条路线：把部分目标转成约束、只优化其余目标；或者把多个目标加权聚合成一个标量损失。无论哪条路线，都指向下一篇要讨论的损失函数设计——本文只需明确：多目标权衡是模拟优化的固有属性，而不是可以回避的细节。

## 7. 一个简化的建模示例

把前面各层拼起来，一个最小的优化问题可以写成如下形式（约束优化）：

```text
minimize    L(x)
subject to  Gain(x)  ≥ Gain_target
            GBW(x)   ≥ GBW_target
            PM(x)    ≥ PM_target
            SR(x)    ≥ SR_target
            Power(x) ≤ Power_max
            x ∈ Ω
```

其中 `L(x)` 是损失函数，约束来自性能指标、工艺规则和工程经验，`Ω` 是前面构造的搜索空间。这个形式背后的观念转变值得强调：

> 对优化器而言，模拟电路设计不再是"调 MOS 管尺寸"，而是在高维约束空间中寻找使性能指标满足目标的设计点。

在代码层面，这个问题被拆成三个解耦的部分：拓扑、参数、搜索空间。

```python
# 仅为示意：拓扑固定，优化器只在 search_space 上搜索

topology = {
    "devices": [
        {"name": "M1", "type": "nmos",
         "nodes": {"D": "voutn", "G": "vinp", "S": "tail", "B": "vss"}},
        {"name": "M2", "type": "nmos",
         "nodes": {"D": "voutp", "G": "vinn", "S": "tail", "B": "vss"}},
    ],
    "nets": ["vinp", "vinn", "voutp", "voutn", "tail", "vdd", "vss"],
}

parameters = {
    "M1": {"w": 10e-6, "l": 180e-9, "m": 2},
    "M2": {"w": 10e-6, "l": 180e-9, "m": 2},   # 与 M1 绑定，保持差分匹配
}

search_space = {
    "w_in": {"type": "float", "low": 1e-6,   "high": 50e-6, "log": True},
    "l_in": {"type": "float", "low": 180e-9, "high": 1e-6,  "log": False},
    "m_in": {"type": "int",   "low": 1,      "high": 16},
    "cc":   {"type": "float", "low": 100e-15,"high": 5e-12, "log": True},
}
```

其中 `topology` 描述连接关系并保持固定，`parameters` 是某个具体设计点的尺寸，`search_space` 是优化器被允许探索的范围。真实工程里，这三者之外还要接上 PDK 器件映射、testbench、仿真配置和指标提取模块，才能构成完整的黑箱 `F(x)`——但优化建模所需的核心抽象，就是这三块加上黑箱函数与约束。

## 8. 在模拟 IC 自动化设计流程中的作用

把本文放回整个系列，它承担的是"建模层"，位于执行流程与优化算法之间：

```text
Virtuoso 自动化接口          (第 1 篇)
        ↓
SKILL 自动生成 schematic      (第 2 篇)
        ↓
Spectre/OCEAN 自动仿真        (第 3 篇)
        ↓
Python 提取性能指标           (第 4 篇)
        ↓
——— 建模层：把设计写成 min L(x) s.t. g(x)≤0 ———  (本篇)
        ↓
优化算法 + 损失函数           (后续)
```

前四篇提供的能力，本质上就是把黑箱 `F(x)` 变得"可调用、可评价、可追踪"。本文则把这个可调用的黑箱，正式包装成一个优化问题：明确了搜索空间 `Ω`、黑箱函数 `F`、指标向量 `m`、约束 `g` 和损失 `L` 的位置。有了这层建模，优化算法才有确定的作用对象——它面对的不再是"一堆脚本"，而是一个定义清晰的数学问题。

## 9. 工程实践中的注意事项

- **不要一上来就把所有器件参数都设成独立变量**。参数维度直接决定搜索空间体积，维度膨胀会让仿真预算被迅速稀释。
- **对称器件必须绑定参数**。差分对、电流镜、负载对应共享变量，既保证匹配，又降低维度。
- **搜索空间要结合 PDK 和版图可实现性设置**。边界应落在工艺允许、版图能画出来的范围内。
- **W/L 不应出现过多无意义的小数位**。可以对连续变量做量化或步进，贴近实际可实现的尺寸。
- **multiplier 和 finger number 用整数变量**，不要当成连续量再四舍五入，否则优化器的分布建模会失真。
- **拓扑固定时，优化器只调 sizing，不改连接关系**。这是保证每个候选点都可仿真的前提。
- **若确实要优化拓扑，应作为更高层的结构搜索问题单独处理**，与 sizing 优化分层，不要混在同一个连续空间里。
- **每次仿真失败都要记录，而不是简单忽略**。失败点本身是可行域边界的重要信息，也是排查参数空间设置问题的线索。
- **黑箱评价成本高，需要比暴力扫描更高效的算法**。高维空间里网格扫描和纯随机搜索会浪费大量仿真，这正是下一篇引入 TPE 的动机。
- **search space、metrics、loss function 尽量解耦**。三者独立，才能在不动其它模块的前提下自由替换优化算法或调整优化目标。

## 10. 小结

本文完成了系列文章中最关键的一次视角转换：把依赖经验和手工调参的模拟电路设计，重写成一个数学优化问题。为此，我们先把电路拓扑抽象成图 `G=(V,E)`，用关联矩阵刻画器件 terminal 与 net 的连接，并把尺寸单独收进参数矩阵，从而实现拓扑固定、sizing 可变；再把设计变量组织成带类型和边界的多维参数空间 `Ω`，其中编码了工艺规则和匹配约束；然后把"改参数—仿真—提指标"的整条链路封装成黑箱函数 `m = F(x)`；最后把性能指标与约束表述成带可行域的约束优化问题 `min L(x) s.t. g(x)≤0`。

贯穿全文的一条主线是解耦：拓扑与 sizing 解耦、参数空间与黑箱函数解耦、指标与损失解耦。正是这些解耦，让模拟设计从"手工艺"变成了一个结构清晰、可被算法反复求解的对象。

本文完成了从模拟电路拓扑到黑箱优化问题的数学建模。到这里，电路设计已经被转化为一个高维、有约束、仿真代价昂贵的搜索问题。下一步，需要选择合适的优化算法在该空间中高效寻找可行解。因此，下一篇文章将介绍基于 TPE 的模拟电路参数优化方法，并讨论它为什么比简单网格扫描和随机搜索更适合高维模拟电路优化。
